# BigData_analysis_of_transport_in_Ukraine_SHI33

![Обраний датасет](https://data.gov.ua/dataset/06779371-308f-42d7-895e-5a39833375f0) - Реєстр зареєстрованих тз Україна


## 1. Ознаки у датасеті
Датасет містить 20 основних ознак, що описують реєстраційні дії та характеристики транспортних засобів (ТЗ):
ʼʼʼ
PERSON: Тип особи (фізична "P" або юридична "J").

REG_ADDR_KOATUU: Код адреси реєстрації за КОАТУУ.

OPER_CODE: Код операції (наприклад, 315 — перереєстрація, 100 — первинна реєстрація).

OPER_NAME: Опис реєстраційної дії.

D_REG: Дата реєстрації.

DEP_CODE: Код сервісного центру (ТСЦ).

DEP: Назва сервісного центру.

BRAND: Марка автомобіля.

MODEL: Модель транспортного засобу.

VIN: Ідентифікаційний номер ТЗ (прихований або частковий у вибірці).

MAKE_YEAR: Рік випуску.

COLOR: Колір ТЗ.

KIND: Тип ТЗ (легковий, вантажний, мотоцикл тощо).

BODY: Тип кузова (універсал, седан, хетчбек тощо).

PURPOSE: Призначення (загальний, спеціалізований).

FUEL: Тип пального (бензин, дизель, електро, газ тощо).

CAPACITY: Об'єм двигуна.

OWN_WEIGHT: Маса ТЗ без навантаження.

TOTAL_WEIGHT: Повна дозволена маса ТЗ.

N_REG_NEW: Новий державний номерний знак.
ʼʼʼ

## 2. Обсяг датасету

Файлова структура: Датасет представлений у форматі CSV.

Оцінка обсягу: Виходячи з наданих фрагментів, файл містить дані про реєстрації за січень 2026 року (з 01.01.26 по 30.01.26) і цей реєстр різний для кожного року. У фрагментах зафіксовано понад 100 000 текстових токенів лише у вибірці. Це свідчить про те, що повний файл може містити сотні тисяч записів, що робить його придатним для Big Data аналізу.

## 3. Завдання для Big Data аналізу
Завдяки різноманітності ознак, цей датасет можна використовувати для вирішення таких завдань:

# 1. Прогнозування попиту на авторинку
Це завдання спрямоване на розуміння вподобань споживачів у розрізі часу та простору.

Основні ознаки: BRAND (Марка), MODEL (Модель), MAKE_YEAR (Рік випуску), REG_ADDR_KOATUU (Регіональний код).

Допоміжні ознаки: D_REG (Дата реєстрації — для виявлення сезонності попиту), BODY (Тип кузова — для аналізу популярності класів, наприклад, SUV чи седани).

# 2. Моніторинг екологічних трендів
Аналіз темпів "електрифікації" автопарку порівняно з традиційними ДВЗ.

Основні ознаки: FUEL (Тип пального: ЕЛЕКТРО, БЕНЗИН, ДИЗЕЛЬ, ГАЗ тощо).

Допоміжні ознаки: D_REG (для побудови часових рядів та динаміки змін), MAKE_YEAR (щоб зрозуміти, чи переходять на електромобілі покупці нових авто, чи вживаних), KIND (щоб розрізняти легкові авто від комерційного транспорту).

# 3. Геопростовий аналіз
Визначення "густоти" автопарку та його специфіки в різних частинах країни.

Основні ознаки: REG_ADDR_KOATUU (Географічний код власника), DEP або DEP_CODE (Локація сервісного центру).

Допоміжні ознаки: KIND та BODY (для розуміння, де більше вантажівок — аграрні регіони, а де легкових — великі міста), TOTAL_WEIGHT (аналіз навантаження на дорожню мережу регіону).

# 4. Кластеризація автопарку (Сегментація)
Групування подібних об'єктів для маркетингового або технічного аналізу.

Основні ознаки: OWN_WEIGHT (Вага), CAPACITY (Об'єм двигуна), BODY (Тип кузова), TOTAL_WEIGHT (Повна вага).

Допоміжні ознаки: KIND (Тип ТЗ), PURPOSE (Призначення — загальний/спеціальний), MAKE_YEAR.

# 5. Аналіз реєстраційної активності
Оптимізація роботи сервісних центрів та розуміння структури ринку.

Основні ознаки: OPER_NAME / OPER_CODE (Тип операції), D_REG (Час операції), DEP (Місце операції).

Допоміжні ознаки: PERSON (Приватні особи чи бізнес генерують черги), N_REG_NEW (Кількість виданих нових номерів).

# 6. Виявлення аномалій (Fraud Detection & Data Cleaning)
Пошук помилок у даних або підозрілих схем.

Основні ознаки: MAKE_YEAR (Дуже старі авто — 1900-ті роки або майбутні дати), OWN_WEIGHT у співвідношенні з CAPACITY та BODY (Нереальні технічні характеристики, наприклад, легковик вагою 10 тонн).

Допоміжні ознаки: VIN (Пошук дублікатів або некоректних форматів), OPER_NAME (Незвичні ланцюжки перереєстрацій за короткий проміжок часу).
